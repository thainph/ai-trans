## AI Translator – Chrome 拡張機能

### 概要

AI Translator は、OpenAI モデルを利用して **あらゆる Web ページ上で選択したテキストをすばやく翻訳できる** ブラウザ拡張機能です。  
使い方はとてもシンプルで、テキストを選択して **T** ボタンを押すだけで、その場に翻訳結果が表示されます。

### 主な機能

- **ページ上でのインライン翻訳**: テキストを選択すると、小さな `T` ボタンが選択範囲の近くに表示され、クリックすると翻訳用ポップアップが開きます。
- **自動言語判定**: 文字種に基づいて、英語、日本語、ベトナム語、韓国語、中国語、フランス語、ドイツ語、スペイン語などの言語を自動判定します。
- **多言語対応の翻訳先**: EN / VI / JA / ZH / KO / FR / DE / ES / PT / RU / TH / ID / IT / NL / AR / HI など、主要な言語に対応しています。
- **文体プリセット**: 翻訳スタイルとして `Casual`（カジュアル）、`Polite`（丁寧）、`Business`（ビジネス）から選択できます。
- **コンパクトなポップアップ UI**:
  - 翻訳元 → 翻訳先の言語ペアを表示。
  - ポップアップ内から翻訳先言語を変更可能。
  - 翻訳結果をワンクリックでコピーできる `Copy` ボタン。
- **Chrome のクラウドストレージに設定を保存**:
  - OpenAI の API キー。
  - デフォルトの翻訳スタイル。
  - デフォルトの翻訳先言語。

### 動作概要

1. **コンテンツスクリプト (`content.js`)**
   - ページ上のテキスト選択を監視します。
   - 選択範囲の近くに丸い `T` ボタンを表示します。
   - ユーザーが `T` をクリックすると、翻訳状態を表示するポップアップ（Shadow DOM）を開きます。
   - 選択されたテキストを取得し、言語を自動判定し、`chrome.storage.sync` からスタイル・翻訳先言語の設定を読み込みます。
   - `text`, `sourceLang`, `targetLang`, `style` を含む `translate` メッセージを **バックグラウンドスクリプト** に送信します。

2. **バックグラウンドサービスワーカー (`background.js`)**
   - コンテンツスクリプトからの `translate` メッセージを受信します。
   - `chrome.storage.sync` から `apiKey` を取得します。
   - 以下に基づいて **system prompt** を生成します:
     - 翻訳元言語と翻訳先言語。
     - 選択されたスタイル（casual / polite / business）。
   - `https://api.openai.com/v1/chat/completions` 経由で OpenAI Chat Completions API（`gpt-4o-mini`）を呼び出します。
   - 翻訳結果のテキストをコンテンツスクリプトに返します。

3. **設定用ポップアップ (`popup.html`, `popup.js`)**
   - ブラウザのツールバーにある拡張機能アイコンをクリックしたときに表示される UI です。
   - 以下の設定が可能です:
     - **OpenAI API キー** の入力・保存。
     - デフォルトの翻訳スタイルの選択。
     - デフォルトの翻訳先言語の選択。
   - 設定は `chrome.storage.sync` に保存され、ページ上のポップアップおよびバックグラウンドスクリプトの両方で利用されます。

### インストールと使い方

#### 1. アンパックド拡張機能としてインストール

1. このリポジトリをクローンまたはダウンロードします。
2. Chrome（もしくは Edge / Brave などの Chromium 系ブラウザ）を開きます。
3. アドレスバーに `chrome://extensions/` と入力して開きます。
4. 右上の **デベロッパーモード** を有効にします。
5. **「パッケージ化されていない拡張機能を読み込む」**（Load unpacked）ボタンをクリックし、`ai-translator` フォルダを選択します。

#### 2. OpenAI API キーの設定

1. ブラウザのツールバーに表示されている **AI Translator** のアイコンをクリックします。
2. 表示されたポップアップで:
   - `API key` フィールドに **OpenAI API キー** を貼り付けます。
   - **Save** ボタンをクリックして保存します。
3. 必要に応じて:
   - デフォルトの翻訳スタイル（`Casual` / `Polite` / `Business`）を選択します。
   - デフォルトの翻訳先言語（例: `Japanese`, `Vietnamese` など）を選択します。

> 注意: API キーはブラウザ内の `chrome.storage.sync` に保存され、ソースコードには含まれません。  
> このリポジトリを共有する際は、API キーをいかなるファイルにも書き込まないようにしてください。

#### 3. Web ページ上でテキストを翻訳する

1. 翻訳したいテキストを含む任意の Web ページを開きます。
2. 翻訳したいテキストを **ドラッグして選択** します。
3. 選択範囲の下に小さな `T` ボタンが表示されるので、クリックして翻訳ポップアップを開きます。
4. ポップアップ内で:
   - 検出された翻訳元言語（例: `EN` → `JA`）が表示されます。
   - ドロップダウンから翻訳先言語を変更できます。
   - 必要に応じてスタイルを変更すると、その設定が自動的に保存されます。
5. 翻訳が完了すると:
   - 結果エリアに翻訳テキストが表示されます。
   - **Copy** ボタンをクリックすることで、翻訳結果をクリップボードにコピーできます。

### プロジェクト構成

- **`manifest.json`**: Manifest V3 の定義ファイル。`storage` / `activeTab` / `https://api.openai.com/*` への `host_permissions`、バックグラウンドサービスワーカー、コンテンツスクリプト、ポップアップ、アイコンなどを宣言します。
- **`background.js`**: OpenAI API を呼び出し、`translate` メッセージを処理します。
- **`content.js`**: UI を挿入し、テキスト選択イベントを監視して `T` ボタンと翻訳ポップアップを表示し、バックグラウンドスクリプトと通信します。
- **`content.css`**: トリガーボタンおよび翻訳ポップアップ（コンテンツスクリプト側）のスタイル。
- **`popup.html`**: 拡張機能アイコンをクリックしたときに表示される設定ポップアップのマークアップ。
- **`popup.js`**: ポップアップ内で API キー、スタイル、翻訳先言語を読み込み・保存するロジック。
- **`popup.css`**: 設定用ポップアップのスタイル。
- **`icons/`**: 拡張機能で使用する 16 / 48 / 128 px のアイコン。

### 権限とセキュリティ

- **使用している Chrome 権限**:
  - `storage`: ブラウザ内に API キー、翻訳スタイル、翻訳先言語を保存するために使用します。
  - `activeTab`: アクティブなタブ上でコンテンツスクリプトを実行し、選択されたテキストを取得するために使用します。
  - `host_permissions`（`https://api.openai.com/*`）: OpenAI API にリクエストを送るために必要です。
- **API キーのセキュリティ**:
  - API キーはブラウザの `chrome.storage.sync` にのみ保存されます。
  - リポジトリ内のどの設定ファイルにも、API キーはハードコードされていません。
  - 他のユーザーがこのプロジェクトを利用する場合は、それぞれが自分の API キーを設定する必要があります。

### 開発ガイド

- 開発環境:
  - バンドラやビルドステップのない、プレーンな JavaScript 構成です。
  - バックグラウンドスクリプトは Manifest V3 のサービスワーカーとして動作します。
- 開発フロー:
  - JS / CSS / HTML ファイルを編集します。
  - `chrome://extensions/` で対象の拡張機能の **Reload** ボタンをクリックします。
  - Web ページタブをリロードして、更新されたコンテンツスクリプトを反映させます。

### 補足

- デフォルトで使用しているモデルは **`gpt-4o-mini`** です。  
- 必要に応じて、`background.js` 内でモデル名や API パラメータを変更できます。
